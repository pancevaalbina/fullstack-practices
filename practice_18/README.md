# Контрольная работа №3 (Практические работы 13–18)

---

## О проекте

Данный репозиторий содержит итоговый проект, объединяющий выполнение практических заданий №13–18. Проект представляет собой полноценное Progressive Web App (PWA) для управления заметками и задачами. Приложение поддерживает работу в офлайн-режиме, синхронизацию в реальном времени и отложенные Push-уведомления.

##Реализованный функционал (по этапам)

* Практика 13 (Service Worker): Реализован базовый офлайн-доступ. Статические ресурсы кэшируются (Cache-first), а данные заметок сохраняются в localStorage.
* Практика 14 (Web App Manifest): Приложение преобразовано в PWA. Добавлен файл manifest.json и набор иконок для установки приложения на домашний экран ПК и мобильных устройств.
* Практика 15 (HTTPS + App Shell): Настроено защищенное соединение через локальные сертификаты (`mkcert`). Внедрена архитектура App Shell для мгновенной отрисовки каркаса интерфейса при плохом интернет-соединении (Network-first для контента).
* Практика 16 (WebSocket + Push): Разработан Node.js сервер. Интегрирована библиотека Socket.io для двусторонней связи (уведомления в реальном времени у всех открытых клиентов при добавлении задачи). Настроена базовая подписка на системные Push-уведомления (`web-push`).
* Практика 17 (Детализация Push): Внедрена система отложенных напоминаний. При добавлении заметки можно указать дату и время срабатывания. Реализована обработка действий (`actions`) в Service Worker: добавлена кнопка «Отложить на 5 минут» (Snooze), переносящая таймер на стороне сервера.
* Практика 18 (Сдача КР №3): Финальная интеграция кода, тестирование работоспособности, подготовка документации (данный README) и публикация репозитория.

## Стек технологий

* Frontend: HTML5, CSS (фреймворк Chota), JavaScript (ES6+), Service Workers, Web App Manifest.
* Backend: Node.js, Express, Socket.io (WebSocket), web-push, cors, body-parser.
* Инструменты: mkcert (локальные SSL-сертификаты), http-server (раздача статики по HTTPS).
